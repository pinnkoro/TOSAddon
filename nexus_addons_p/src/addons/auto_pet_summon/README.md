# Auto Pet Summon

> キャラ毎に最後に連れていたペットを自動で召喚します

| 項目 | 内容 |
| --- | --- |
| キー | `auto_pet_summon` |
| ソース | [auto_pet_summon.lua](auto_pet_summon.lua) |
| 設定画面 | なし（アドオン一覧の ON/OFF のみ） |

## 使い方

1. ON にする。
2. 普段どおりペットを召喚する。**その時点で「このキャラのペット」として自動的に記録されます**
   （登録操作は不要）。
3. 以降、そのキャラで**街に入るたび**に、記録したペットが自動で召喚されます。

ペットを手動でしまうと記録も消えます（次から自動召喚しなくなります）。
連れ替えたい場合は、単に別のペットを召喚し直してください。

まだ一度もペットを出していないキャラで街に入ると、チャットに
`[APS] <キャラ名> ペット未登録` と出ます。1 回召喚すれば以降は出ません。

## しくみ

* 記録は**キャラクター（CID）単位**で、`{ iesid, clsid }` の組を持ちます。
* 発火は「マップ種別が `City`」かつ「前回処理時から CID が変わった」ときだけです。
  つまり**街に入ったとき／キャラチェンジ直後**に 1 回だけ動きます。
* 召喚後は 1.0 秒間隔のタイマーで「実際に召喚中か」と「記録があるか」を比較し、
  食い違ったときだけ記録を書き換えて保存します（毎秒書き込むわけではありません）。

## 保存先

`../addons/_nexus_addons_p/<アカウントID>/auto_pet_summon.json`

```json
{ "<CID>": { "iesid": "<ペットのGUID>", "clsid": <ペットのClassID> } }
```

`clsid` が `0` のときは「未登録」の意味です。

## 注意

* **街（`City`）でしか動きません。**フィールドやダンジョンでは自動召喚しません。
* 本家 Nexus Addons など、同名の旧初期化関数（`AUTO_PET_SUMMON_ON_INIT`）を持つ
  アドオンが読み込まれている場合は、競合を避けるため自動召喚をスキップします。
* 派生版では、初めて使うキャラへキャラチェンジしたときに設定エントリが作られず
  自動召喚が動かない不具合、および街に入るたび設定ファイルを読み直していた不具合を
  修正済みです（v1.0.1）。
