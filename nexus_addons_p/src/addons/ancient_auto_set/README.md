# Ancient Auto Set

> アシスターセットをキャラ毎に自動で付け替えます（事前にアシスター保管箱で設定必要）

| 項目 | 内容 |
| --- | --- |
| キー | `ancient_auto_set` |
| ソース | [ancient_auto_set.lua](ancient_auto_set.lua) |
| 設定画面 | なし（アシスター保管箱に設定用のボタンとプリセット画面が追加されます） |

## 使い方

### 1. キャラごとのセットを登録する

1. ON にして**アシスター保管箱（`ancient_card_list`）**を開く。
2. そのキャラで使いたいアシスターを 4 枠にセットする。
3. 画面内に追加された**歯車ボタン**を **左クリック**する → `[AAS]登録しました`

以降、そのキャラで**街に入るたび**に、登録した 4 枚が自動でセットされます
（すでに一致していれば何もしません）。

歯車ボタンを**右クリック**すると `[AAS]登録解除しました` となり、自動セットが止まります。

未登録のキャラで街に入ると `[AAS] <キャラ名> アシスター未登録` と表示されます。

### 2. プリセット（10 セット）を使う

アシスター保管箱を開くと、右側に **`Assister Preset Setting`** ウィンドウが一緒に出ます。

| 操作 | 動作 |
| --- | --- |
| `Set 1`〜`Set 10` タブ | 編集するプリセットを切り替える |
| アシスターを枠へ**ドラッグ＆ドロップ** | そのプリセットの枠に登録 |
| 枠を**右クリック** | その枠の登録を消す |
| 入力欄に名前を入れて **Enter** | そのプリセットに名前を付ける |
| **`Change` ボタン** | **プリセットの内容を実際に装着する** |

キャラ登録（1.）が「街に入ったら自動で戻す」用途なのに対し、
プリセット（2.）は「その場で手動で 10 種類を切り替える」用途です。

## しくみ

* `ANCIENT_CARD_LIST_OPEN` / `ANCIENT_CARD_LIST_CLOSE` をフックして、
  アシスター保管箱の開閉に合わせてプリセット画面を出し入れします。
* 装着は `ReqSwapAncientCard(GUID, スロット番号)` を **0.3 秒間隔で 1 枠ずつ**、
  枠 0〜3 の順に実行します（まとめて投げると取りこぼすため）。
  現在すでに同じカードが入っている枠は飛ばします。
* `Change` は **`IS_ANCIENT_ENABLE_MAP()` が `YES` を返すマップでは実行できません**
  （`ImpossibleInCurrentMap` を表示して中止）。
* キャラ登録の自動適用は**街（`City`）に入ったときだけ**動きます。

## 保存先

`../addons/_nexus_addons_p/<アカウントID>/ancient_auto_set.json`

```json
{
  "ver": 1.1,
  "priset": { "0": { "name": "...", "0": "<GUID>", "1": "...", "2": "...", "3": "..." } },
  "<CID>":  { "name": "<キャラ名>", "guids": [ ["<GUID>", "<クラス名>"], ... ] }
}
```

初回のみ、旧 `ancient_autoset` アドオンの
`../addons/ancient_autoset/<アカウントID>/settings.json` からキャラ設定とプリセットを引き継ぎます
（引き継いだカードのクラス名は不明なので `Unknown` で入り、次に街へ入ったときに補完されます）。

## 注意

* **登録されるのは「そのとき装備していた 4 枚の GUID」**です。カードを売却・分解すると
  自動セットが失敗します（その枠は変わらないまま次の枠へ進みます）。
* 保管箱を開かずに登録・解除はできません。
* 本家 Nexus Addons など、同名の旧初期化関数（`ANCIENT_AUTOSET_ON_INIT`）を持つ
  アドオンが読み込まれている場合は、競合を避けるため初期化をスキップします。
