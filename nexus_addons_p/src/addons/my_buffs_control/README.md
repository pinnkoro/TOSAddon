# My Buffs Control

> バフ欄を移動可能にして、選択したバフを非表示にします（街では動作しません）

| 項目 | 内容 |
| --- | --- |
| キー | `my_buffs_control` |
| ソース | [my_buffs_control.lua](my_buffs_control.lua) |
| 設定画面 | あり（アドオン一覧の歯車ボタン） |

## 使い方

ON にすると、自分のバフ欄（`buff` フレーム）の左上に**鍵アイコン**が付きます。

| 操作 | 動作 |
| --- | --- |
| 鍵アイコンを左クリック | ロックの ON/OFF。解除するとバフ欄をドラッグで移動できる |
| バフ欄をドラッグ | 移動。離した位置が保存される |

### 表示するバフを選ぶ

アドオン一覧（メニューボタン → アドオン一覧）の歯車ボタンから設定画面を開き、
**「Buff list」ボタン**を押すとバフ一覧が開きます。

* チェックが入っているバフだけが表示されます。**チェックを外すと非表示**になります。
* 検索欄で名前を絞り込めます。
* チェック済みのバフが一覧の上に来るよう並べ替えられます。

設定画面の **「Default Pos」ボタン**でバフ欄の位置を既定（`20, 130`）に戻せます。

## しくみ

* `BUFF_ON_MSG` をフックし、バフが付いた時点で「非表示指定なら `COMMON_BUFF_MSG` に `REMOVE` を
  投げる」ことで、バフ欄から取り除いています。
* 初期化時（およびその 1 秒後）に、現在表示されているバフと実際に付いているバフを突き合わせて
  差分を埋めます。ログイン直後やマップ移動直後に取りこぼしが出ないようにするためです。
* **デバフ（`Group1` が `Debuff` / `Deuff`）は設定に関係なく常に表示**します。
* **セパレートバフフレームに出るバフ**（`BUFF_CHECK_SEPARATELIST`）には手を出しません。
* 位置は「街用（既定位置）」と「街以外用（保存位置）」を分けて持っており、
  街に入ると既定位置に戻ります。

## 保存先

| ファイル | 内容 |
| --- | --- |
| `../addons/_nexus_addons_p/<アカウントID>/my_buffs_control.json` | 位置とロック状態 |
| `../addons/_nexus_addons_p/<アカウントID>/my_buffs_control.dat` | バフごとの表示設定（`<バフID>:::<0/1>` の行） |

```json
{ "lock": true, "default_x": 20, "default_y": 130,
  "custom_x": 20, "custom_y": 130, "ver": 1.2 }
```

バフの設定だけ `.dat` に分けてあるのは件数が多いためです。
初回のみ、旧 `my_buffs` アドオンの `../addons/my_buffs/settings_2503.json` から引き継ぎ、
そこに無いバフは「表示」で埋めます。

## 注意

* **街（`City`）では何もしません。**バフ欄は既定位置に戻り、すべてのバフが表示されます
  （設定画面にも赤字で注意書きが出ます）。
* 新しく出会ったバフは自動的に「表示」として登録されます。そのため、
  **非表示にしたバフでも、街の外でそのバフを受け直すと表示に戻ることがあります。**
* 設定ファイルの `ver` が 1.2 未満のときは、位置とバフ設定をすべて初期化して作り直します。
* 本家 Nexus Addons など、同名の旧初期化関数（`MY_BUFFS_ON_INIT`）を持つ
  アドオンが読み込まれている場合は、競合を避けるため初期化をスキップします。
