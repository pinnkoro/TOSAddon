# Save Quest

> ワープ用のクエスト誤完了防止

| 項目 | 内容 |
| --- | --- |
| キー | `save_quest` |
| ソース | [save_quest.lua](save_quest.lua) |
| 設定画面 | あり（アドオン一覧の歯車ボタン） |
| 原作者 | weizlogy さん |

## 何をするアドオンか

一部のクエストは、受注中であれば**クエストウィンドウからワープできる**ため、移動手段として
残しておく人が多くあります。ところが対象 NPC をうっかりクリックすると完了してしまい、
そのワープ先を失います。

このアドオンは、**保存したクエストの NPC をクライアント側から消す**ことで、
誤って話しかけられないようにします。あわせて、よく使うワープを一覧にまとめる
ショートカットパネルを提供します。

## 使い方

### クエストを保存する（誤完了防止）

1. ON にしてクエストウィンドウを開き、**「進行中」タブ**を表示する。
2. ワープ可能なクエストの行に、回転する**ワープアイコン**が追加されます。
   * **左クリック** … そのクエストのワープ先へ飛ぶ
   * **右クリック** … メニューを開く
3. メニューから `Save` を選ぶと保存され、行に `saved` と表示されます。

保存すると、そのクエストの NPC は**視界に入っても表示されず、ターゲットもできなく**なります。
解除は同じメニューの `Release` です。

| メニュー | 動作 |
| --- | --- |
| `Save` | NPC を非表示にして誤完了を防ぐ |
| `Release` | 保存を解除して NPC を元に戻す |
| `ShortCut` | ショートカットパネルに追加する |
| `ShortCut Release` | ショートカットパネルから外す |
| `Cancel` | 何もしない |

クエストを**放棄すると、保存とショートカットの両方が自動で外れます**。

### ショートカットパネル

`ShortCut` に登録したクエストが、独立した小さなパネルに並びます。1 行につき

| 要素 | 動作 |
| --- | --- |
| ワープアイコン | 左クリックでワープ。右クリックでショートカットメニュー |
| ワープ先のマップ名 | 表示のみ |
| 共有アイコン | クリックで**クエストのパーティー共有**をトグル。共有中は白、非共有は灰色 |

登録が 0 件になるとパネルは 0×0 に潰れて見えなくなります。

## 設定

アドオン一覧（メニューボタン → アドオン一覧）の歯車ボタンから開きます。

| 項目 | 既定値 | 説明 |
| --- | --- | --- |
| チェックするとフレーム固定 | OFF（移動可） | ショートカットパネルのドラッグ移動を禁止する |
| SKIN SELECT | 透明度高め | パネルの背景。`無し` / `黒` / `透明度高め` から選ぶ |

パネルの位置はドラッグして離した時点で保存されます。

## しくみ

* NPC を消すのは `world.Leave(handle, 0.0)`（クライアント側の退場）です。
  * **初期化時**に半径 1000 のオブジェクトを走査して対象を消す
  * 以降は `TARGET_SET`（何かをターゲットしたとき）に判定して消す
* 保存の単位は**クエスト ID ではなく NPC の状態名**（`<状態>NPC` プロパティ）です。
  そのため、同じ NPC を使う別状態には影響しません。
* ワープ可否は `POSSI_WARP` / `PROG_WARP` / `SUCC_WARP` を現在のクエスト状態と突き合わせて判定します。
  ワープできないクエストにはアイコンが付きません。
* `SET_QUEST_CTRL_MARK` / `SCR_QUEST_SHARE_PARTY_MEMBER` / `EXEC_ABANDON_QUEST` をフックしています。

## 保存先

`../addons/_nexus_addons_p/<アカウントID>/save_quest.json`

```json
{
  "ver": 1.1,
  "save_quests": { "<NPC状態名>": 1 },
  "short_cuts": { "<クエストID>": 1 },
  "frame": { "move": 0, "x": 0, "y": 0, "skin": "bg2" }
}
```

`ver` が 1.1 未満の古い設定は、読み込み時に `frame.skin` を `bg2` に補正して更新します。

## 注意

* NPC を消すのは**自分のクライアントの表示だけ**です。サーバー側では NPC は存在しています。
* **アイコンが出るのはクエストウィンドウの「進行中」タブだけ**です。
* 本家 Nexus Addons など、同名の旧初期化関数（`SAVEQUEST_ON_INIT`）を持つ
  アドオンが読み込まれている場合は、競合を避けるため初期化をスキップします。
