# Pick Item Tracker

> アイテム取得情報表示

| 項目 | 内容 |
| --- | --- |
| キー | `pick_item_tracker` |
| ソース | [pick_item_tracker.lua](pick_item_tracker.lua) |
| 設定画面 | なし（フレームの右クリックで位置ロック） |

## 使い方

ON にしてフィールドへ出ると、**そのマップで拾ったアイテムの一覧と滞在時間**が
小さなフレームに表示されます。同じアイテムを拾うたびに個数が加算されます。

| 操作 | 動作 |
| --- | --- |
| **ドラッグ** | フレームを移動（離した位置が保存される） |
| **右クリック** | 位置ロックの ON/OFF（鍵アイコンの明暗で分かる） |

アイテム名はレアリティで色分けされます。

| 色 | レアリティ |
| --- | --- |
| 白 | ノーマル |
| 青 | マジック |
| 紫 | レア |
| 橙 | ユニーク |
| 赤橙 | レジェンド |
| 黄 | 女神 |

タイトルの下には `00:00:00` 形式で**そのマップに入ってからの経過時間**が出ます。

## しくみ

* `ITEM_PICK` を購読し、アイテム名をキーに `{ ClassID, 個数 }` を積み上げます。
* **集計はマップ単位でリセット**されます。`g.map_id` が変わった時点で一覧と計測開始時刻を
  作り直します（同じマップに戻っただけでは消えません。別マップを挟むと消えます）。
* 計測開始時刻は「マップに入った時刻 − 3 秒」です（ロード時間ぶんの補正）。
* 表示は 1.0 秒間隔で経過時間だけ更新し、アイテム行は拾ったときだけ書き換えます。
  新しい種類を拾ったときだけ一覧全体を組み直します。
* フレームは ESC で消えない土台（`g.create_persistent_frame`）で作ります。

## 保存先

`../addons/_nexus_addons_p/<アカウントID>/pick_item_tracker.json`

```json
{ "move": 1, "x": 330, "y": 330 }
```

`move` が `1` で移動可、`0` でロックです。

## 注意

* **街（`City`）とインスタンス（`Instance`）では非表示**になります。
  そこでは集計も行いません。
* 集計は**メモリ上のみ**で、ファイルには残りません。クライアントを落とすと消えます。
  マップごとの永続的な集計が欲しい場合は Monster Kill Count を使ってください。
* 本家 Nexus Addons など、同名の旧初期化関数（`PICK_ITEM_TRACKER_ON_INIT`）を持つ
  アドオンが読み込まれている場合は、競合を避けるため初期化をスキップします。
* 派生版では、ESC キーで表示が消えてしまう不具合を修正済みです（v1.0.1）。
