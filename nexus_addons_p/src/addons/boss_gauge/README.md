# Boss Gauge

> ボスゲージにスタン値とシールド値を表示します

| 項目 | 内容 |
| --- | --- |
| キー | `boss_gauge` |
| ソース | [boss_gauge.lua](boss_gauge.lua) |
| 設定画面 | なし（アドオン一覧の ON/OFF のみ） |

## 使い方

ON にするだけです。ボスをターゲットすると、画面上部のボス情報バー（`targetinfotoboss`）の
名前の右側に、次の 2 つが割合（%）で追加表示されます。

| 表示 | 意味 |
| --- | --- |
| `STUN:(xx.xx%)` | **スタンの蓄積割合**。`(最大faint - 現在faint) / 最大faint`。100% でスタン |
| `SHIELD:(xx.xx%)` | **シールド残量**。最大 HP に対するシールド量の割合 |

`STUN` の値は「溜まった量」です。ボスの `faint` は満タンから減っていくため、
表示上は 0% から増えていき、100% に達したところでスタンします。

## しくみ

* `TARGET_BUFF_UPDATE` / `TARGET_UPDATE` / `TARGET_SET_BOSS` を購読し、
  そのたびに `info.GetStat(session.GetTargetBossHandle())` から `cur_faint` / `max_faint` を読んで
  `stun` リッチテキストを更新します。長いボス名で数値と重ならないよう、
  名前ラベルは幅 220 に収まるようフォントサイズを自動調整します。
* シールドはイベントが無いため `TARGETINFOTOBOSS_UPDATE_SHIELD` を**フック**して
  引数（`シールド量/最大HP` 形式の文字列）を横取りし、`shield_text` を更新します。
* シールドゲージ自体が非表示になったときは `shield_text` も隠します
  （倒しかけのボスに古い数値が残らないようにするため）。

## 保存先

なし（ON/OFF は `../addons/_nexus_addons_p/<アカウントID>/settings.json` に入ります）。

## 注意

* 対象は**ボス扱いのターゲット**のみです。通常モンスターのゲージには何も追加しません。
* 表示位置はボス情報バーの名前ラベル幅から算出した固定オフセットです。
  他アドオンが同じフレームを改造している場合、重なることがあります。
