# Status Point Check

> ステータスポイントがもらえるクエストのクリア状況表示

| 項目 | 内容 |
| --- | --- |
| キー | `status_point_check` |
| ソース | [status_point_check.lua](status_point_check.lua) |
| 設定画面 | あり（アドオン一覧の歯車ボタンで一覧が開きます） |
| 原作者 | torahamu さん |

## 使い方

ON にしてキャラクター情報（ステータス）ウィンドウを開くと、
**1 番目のタブに「クエスト確認」ボタン**が追加されます。押すと一覧ウィンドウが開きます
（アドオン一覧の歯車ボタンからも同じ一覧が開きます）。

一覧は 3 つのタブに分かれています。

| タブ | 内容 |
| --- | --- |
| **Status** | ステータスポイント／各種ステータスがもらえるクエストの達成状況 |
| **Zemina** | ゼミナ像（`statue_zemina`）の起動状況をマップ別に一覧 |
| **Master Quest** | マスタークエスト（報酬に `Point_Stone_100_Q` を含むクエスト）の達成状況 |

各行の色と表記で状態が分かります。

| 表示 | 意味 |
| --- | --- |
| 赤字 `OK` | 達成済み（`SCR_QUEST_CHECK` が `SUCCESS` / `COMPLETE`） |
| 灰字 `NO` | 未達成。右側に**クエスト開始マップと推奨レベル**が出るので、そこへ取りに行けばよい |

行頭の巻物アイコンをクリックすると、そのクエストの詳細（ゲーム標準のクエスト情報）が開きます。

### Status タブの内訳

* 上段 **`Quest List For Status Point (取得済み/合計)`**
  `Success_StatByBonus` が 1 以上のクエスト、つまり**ステータスポイントそのもの**をくれるものです。
  見出しに現在の合計取得ポイントが出ます。各行の右側に `<n> Point` が付きます。
* 下段 **`Quest List For Status`**
  `reward_property` に登録された、ステータスポイント以外の恒常報酬をくれるクエストです。
  各行の右側に `<効果> + <値>`（例: `所持量 + 20`）が付きます。

### Zemina タブ

ゼミナ像が配置されている全マップを走査し、像の NPC 状態が `20` または `1` なら `OK`、
それ以外は `NO` を表示します。

## 保存先

なし（ON/OFF は `../addons/_nexus_addons_p/<アカウントID>/settings.json` に入ります）。

## 注意

* 一覧はゲームのクラスデータを毎回全走査して作るため、**開くのに少し時間がかかります**。
* ボタンが出るのはステータスウィンドウの**最初のタブ**だけです。他のタブに切り替えると消えます。
* 判定はクライアント側の `SCR_QUEST_CHECK` によるものです。表示と実際の受領可否が食い違う場合は
  ゲーム側が正です。
