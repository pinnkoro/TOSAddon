# Auto Repair

> 装備の耐久を監視して自動で緊急修理キットを使用します（女神の証商店からの自動補充付き）

| 項目 | 内容 |
| --- | --- |
| キー | `auto_repair` |
| ソース | [auto_repair.lua](auto_repair.lua) |
| 設定画面 | あり（アドオン一覧の歯車ボタン） |

## 使い方

ON にするだけです。装備の耐久が下がると、インベントリの**緊急修理キット**
（ClassID `11201388`）を自動で使って修理します。

街に入ったとき、修理キットの残りが**お知らせ数量以下**だと

* 「自動補充有効化」が OFF → `修理キットの残りが少ないですが補充しますか？` の確認ダイアログ
* 「自動補充有効化」が ON → `[Auto Repair] 自動補充します` と表示して即座に購入

となり、**女神の証商店**（`AustejaCertificate`）から自動購入数まで買い足します。
購入数は `自動購入数 − 現在の所持数` です。

## 設定

アドオン一覧（メニューボタン → アドオン一覧）の歯車ボタンから開きます。

| 項目 | 既定値 | 説明 |
| --- | --- | --- |
| 自動購入数入力 | `50` | 補充後の目標所持数。**Enter キーで確定** |
| 入力数以下で補充メッセージ | `20` | この数以下になったら街で通知する。**Enter キーで確定** |
| 自動補充有効化 | OFF | ON にすると確認なしで購入する |

## しくみ

* ゲーム側の耐久通知（`DURNOTIFY_UPDATE`）を**置き換え**て、通知フレームの中身を自前で描き直します。
  * 装備一覧を走査し、`IS_DUR_UNDER_10PER` が真の装備だけをアイコンで並べる
  * 耐久 0 のものは赤（`FF990000`）、それ以外は黄（`FF999900`）で着色する
  * 状態に応じて `DurUnder30` / `DurUnder0` のシステムメッセージを 1 回だけ出す
* 耐久が下がった装備を見つけた時点で 1 度だけ修理キットを使います
  （緊急修理キットは全装備をまとめて直すため、1 回の通知につき 1 系統で足ります）。
  対象の耐久が最大値の 90% 未満である限り、**所持数と 4 回の少ない方**まで繰り返します。
* **ロック中の修理キットは使いません。**
* 初期化時に耐久通知フレーム（`durnotify`）が表示されていた場合は 0×0 に潰します。

## 保存先

`../addons/_nexus_addons_p/<アカウントID>/auto_repair.json`

```json
{ "buy_qty": 50, "msg_qty": 20, "setting_x": 800, "setting_y": 700, "auto_buy": false }
```

## 注意

* **補充の確認／自動購入は街（`City`）に入ったときだけ**行われます。
  フィールドで切らしても購入はしません。
* 自動補充には**女神の証**が必要です。足りない場合はゲーム側の取引が失敗します。
* 本家 Nexus Addons など、同名の旧初期化関数（`AUTO_REPAIR_ON_INIT`）を持つ
  アドオンが読み込まれている場合は、競合を避けるため初期化をスキップします。
* 派生版では、修理キットのアイテム ID を Lv.540 用から **Lv.550 用へ修正**済みです。
  これが古いままだとキットが検出されず、街に入るたびに購入を繰り返していました（v1.0.0）。
